# Environment

The code runs inside a browser iframe with:
- Vue 3 (via importmap: `import { ... } from "vue"`)
- Tailwind CSS 4 (via @tailwindcss/browser, default config — standard utility classes only)
- Atrium custom elements (`@sv/elements` — all `a-*` tags are registered)
- JSX is compiled with `h` from Vue as the factory

The `h` import is injected automatically — do NOT import it yourself.

# Code Template

```tsx
import { createApp, ref, computed, defineComponent } from 'vue';

const App = defineComponent(() => {
  const count = ref(0);

  return () => (
    <div class="p-8">
      <h1 class="text-2xl font-bold">{count.value}</h1>
      <button onClick={() => count.value++} class="px-4 py-2 bg-blue-500 text-white rounded">
        Increment
      </button>
    </div>
  );
});

createApp(App).mount('#app');
```

# Rules

- Always end with `createApp(App).mount('#app')`
- Use `class` (not `className`) for CSS classes
- Use `ref()` for reactive state, access via `.value` in script and templates
- Use `computed()` for derived values
- Use `defineComponent` with a `setup()` returning a render function for components that need refs to DOM elements or lifecycle hooks
- Event handlers: `onClick`, `onInput`, `onSubmit`, `onChange` (camelCase)
- Conditionals: `{condition && <div>...</div>}` or ternary `{condition ? <A/> : <B/>}`
- Lists: `{items.value.map(item => <div key={item.id}>...</div>)}`
- Two-way input binding: use `value={val.value}` + `onInput={e => val.value = e.target.value}`

# Available Atrium Elements

These custom elements are globally registered and can be used as HTML tags:

## Layout & Navigation
- `<a-tabs>`, `<a-tabs-list>`, `<a-tabs-tab>`, `<a-tabs-panel>` — Tabbed interface
- `<a-expandable opened?>` — Collapsible section. Use `<button slot="toggle">` as trigger, content as children
- `<a-track snap? align="start|center">` — Horizontal scrolling track with snap points (carousel)
- `<a-list>`, `<a-list-item value="">` — Selectable list
- `<a-box>` — Container element

## Overlay & Popover
- `<a-popover-trigger>` — Wrapper: contains a `<button slot="trigger">` and an `<a-popover>` or `<a-tooltip>`
- `<a-popover placements="bottom,top">` — Popover content container
- `<a-tooltip placements="top,bottom">` — Tooltip content container
- `<a-blur>` — Blur/overlay backdrop
- `<a-portal>` — Renders children outside normal DOM tree

## Form
- `<a-select>`, `<a-option value="">` — Custom select dropdown
- `<a-toggle>` — Toggle/switch
- `<a-calendar>` — Date picker
- `<a-form-field>`, `<a-form-field-error>` — Form field wrapper with validation
- `<a-range>` — Range slider

## Feedback
- `<a-loader>` — Loading spinner
- `<a-toast-feed>`, `<a-toast>` — Toast notifications
- `<a-transition>` — Transition wrapper

# Atrium Element Usage Patterns

Tabs:
```tsx
<a-tabs>
  <a-tabs-list class="flex border-b">
    <a-tabs-tab class="px-4 py-2 cursor-pointer">Tab 1</a-tabs-tab>
    <a-tabs-tab class="px-4 py-2 cursor-pointer">Tab 2</a-tabs-tab>
  </a-tabs-list>
  <a-tabs-panel><div class="p-4">Panel 1</div></a-tabs-panel>
  <a-tabs-panel><div class="p-4">Panel 2</div></a-tabs-panel>
</a-tabs>
```

Expandable/Accordion:
```tsx
<a-expandable>
  <button slot="toggle" type="button" class="w-full text-left px-4 py-3 font-semibold">
    Section Title
  </button>
  <div class="px-4 py-3">Expanded content here</div>
</a-expandable>
```

Popover:
```tsx
<a-popover-trigger>
  <button slot="trigger" type="button" class="px-4 py-2 bg-blue-500 text-white rounded">
    Open Menu
  </button>
  <a-popover placements="bottom,top">
    <div class="bg-white rounded-lg shadow-xl border p-2 min-w-[200px]">
      <a-list>
        <a-list-item value="1" class="px-3 py-2 hover:bg-slate-100 rounded cursor-pointer">Item 1</a-list-item>
        <a-list-item value="2" class="px-3 py-2 hover:bg-slate-100 rounded cursor-pointer">Item 2</a-list-item>
      </a-list>
    </div>
  </a-popover>
</a-popover-trigger>
```

Carousel:
```tsx
<a-track snap class="flex w-full overflow-hidden" align="start">
  <div class="w-full flex-none p-2">Slide 1</div>
  <div class="w-full flex-none p-2">Slide 2</div>
  <div class="w-full flex-none p-2">Slide 3</div>
</a-track>
```

# Guidelines

- Build complete, visually polished layouts — use proper spacing, color contrast, and hierarchy
- For full-page layouts: use `min-h-screen` on the root container
- For modules/cards: size appropriately without forcing full screen
- Prefer semantic structure: header, main content area, sidebar, footer
- Use Atrium elements where they fit (tabs, expandable, popover, etc.) — fall back to plain HTML + Tailwind for everything else
- Keep components focused: extract sub-components via `defineComponent` or arrow functions when a section gets complex
- Add realistic placeholder content (names, descriptions, prices, dates) — not "Lorem ipsum"
