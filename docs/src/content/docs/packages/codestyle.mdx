This package includes confugration files for different tools used to enforce a consistent code style accross projects.

## Installation

Make sure your project is set up with access to the registry as described in [Setup](/atrium/setup).

<Command>
  <div slot="pnpm">
    ```sh
    pnpm add @svp/codestyle --save-exact
    ```

  </div>
  <div slot="npm">

## Manual setup

### Biome

:::cation
For the biome configuration to work with your editior, the `biome.json` file needs to be in the root of the project.
:::

```json
// biome.json
{
  "$schema": "./node_modules/@biomejs/biome/configuration_schema.json",
  "extends": ["@svp/codestyle/biome"]
}
```

### Prettier

We should use Biome for js formatting but Prettier is still useful for many other files and integrations.

```json
// package.json
{
  "name": "your-package",
  ...
  "prettier": "@svp/codestyle/.prettierrc.json"
}
```

Prettier [docs](https://prettier.io/docs/en/configuration.html).

### TypeScript

```json
// tsconfig.json
{
  "$schema": "http://json.schemastore.org/tsconfig",
  "extends": ["@svp/codestyle/tsconfig.json"]
}
```

### Commitlint

```javascript
// commitlint.config.js
module.exports = {
  extends: ["@svp/codestyle/commitlint.cjs"],
};
```

<br/>

:::note
Reccomended for a new setup is biome instead of eslint and stylelint.
:::

### ESLint

```json
// .eslintrc.json
{
  "extends": ["@svp/codestyle/.eslintrc.json"]
}
```

### Stylelint

```javascript
// stylelint.config.cjs
module.exports = {
  extends: ["@svp/codestyle/stylelint.config.cjs"],
};
```
