## Installation

Make sure your project is set up with access to the registry as described in [Usage](/sv-frontend-library/mono/usage#configure-package-registry).

<Command>
  <div slot="pnpm">
    ```sh
    pnpm add @sv/codestyle --save-exact
    ```
  </div>
  <div slot="npm">
    ```sh
    npm i @sv/codestyle --save-exact
    ```
  </div>
  <div slot="bun">
    ```sh
    bun add @sv/codestyle --exact
    ```
  </div>
</Command>

## Usage

### Editorconfig

The `.editorconfig` will be copied next to your package.json using a [postinstall script](https://docs.npmjs.com/cli/v10/using-npm/scripts#npm-install) -- if not already present.

### Biome

:::cation
For the biome configuration to work with your editior, the `biome.json` file needs to be in the root of the project.
:::

```json
// biome.json
{
  "$schema": "./node_modules/@biomejs/biome/configuration_schema.json",
  "extends": ["./node_modules/@sv/codestyle/biome.json"]
}
```

### Prettier

```json
// .prettierrc.json
{
  "extends": "@sv/codestyle/.prettierrc.json"
}
```

### Commitlint

```javascript
// commitlint.config.js
module.exports = {
  extends: ["@sv/codestyle/commitlint.cjs"],
};
```

### ESLint

```json
// .eslintrc.json
{
  "extends": ["@sv/codestyle/.eslintrc.json"]
}
```

### Stylelint

```javascript
// stylelint.config.cjs
module.exports = {
  extends: ["@sv/codestyle/stylelint.config.cjs"],
};
```

### TypeScript

```json
// tsconfig.json
{
  "$schema": "http://json.schemastore.org/tsconfig",
  "extends": ["@sv/codestyle/tsconfig.json"]
}
```
